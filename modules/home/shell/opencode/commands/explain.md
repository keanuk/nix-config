Explain code, a file, or a concept in the project clearly and concisely.

Process:
1. Identify what the user wants explained — a specific function, file, module, or concept
2. Read the relevant code thoroughly
3. Provide a high-level summary first, then details
4. Use concrete references (function names, line numbers, file paths)

For a file or module:
- What is its single responsibility?
- What are its public interfaces (functions, exports, options)?
- What are its key dependencies?
- Any non-obvious invariants or assumptions?

For a function or code block:
- What does it do in plain language?
- What are its inputs and outputs?
- Are there side effects or hidden dependencies?
- Any edge cases or error handling worth noting?

Style:
- Start with the simplest accurate description, then add detail if needed
- Use analogies sparingly and only when they genuinely clarify
- Flag anything that looks like a bug, workaround, or technical debt
- If the code is complex, suggest ways it could be simplified
